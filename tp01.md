# TP1 — Productos Reales con Machine Learning

## 1. Amazon - Recomendaciones de Productos
- **Nombre del producto/servicio**: Sistema de recomendaciones de Amazon.
- **Qué problema resuelve con ML**: Personaliza sugerencias de productos para mejorar la experiencia de compra, aumentando las ventas al predecir items que los usuarios podrían querer basado en su comportamiento pasado.
- **Qué tipo de ML usa y por qué**: Supervisado, porque se entrena con datos etiquetados como historiales de compras y visualizaciones para clasificar y recomendar productos similares. Utiliza modelos como filtrado colaborativo, que aprenden de ejemplos conocidos para hacer predicciones precisas.
- **Qué datos probablemente necesita para funcionar**: Historial de navegación y compras de usuarios, metadatos de productos (categorías, reseñas, precios), y datos de interacción como clics o tiempo en página.

## 2. Google Photos - Reconocimiento de Imágenes
- **Nombre del producto/servicio**: Funcionalidad de búsqueda y organización en Google Photos.
- **Qué problema resuelve con ML**: Organiza automáticamente fotos y permite búsquedas por contenido visual, resolviendo la dificultad de manejar grandes bibliotecas de imágenes al identificar objetos, personas y escenas.
- **Qué tipo de ML usa y por qué**: Supervisado, ya que emplea redes neuronales convolucionales (CNN) entrenadas con imágenes etiquetadas para clasificar y detectar elementos en fotos nuevas. Es supervisado porque usa datos con etiquetas (ej. "perro", "montaña") para aprender patrones.
- **Qué datos probablemente necesita para funcionar**: Imágenes subidas por usuarios, etiquetas de entrenamiento de datasets como ImageNet, metadatos de fotos (ubicación, fecha), y retroalimentación de usuarios para mejorar el modelo.

## 3. IBM Watson Health - Diagnóstico Médico
- **Nombre del producto/servicio**: IBM Watson para atención médica (ej. Watson for Oncology).
- **Qué problema resuelve con ML**: Asiste a médicos en diagnósticos y tratamientos personalizados, analizando datos médicos complejos para identificar patrones y sugerir opciones basadas en evidencia.
- **Qué tipo de ML usa y por qué**: Supervisado, porque se entrena con datos médicos etiquetados como historiales de pacientes y resultados de tratamientos para predecir outcomes. Utiliza procesamiento de lenguaje natural (NLP) y clasificación para aprender de casos conocidos.
- **Qué datos probablemente necesita para funcionar**: Registros médicos anonimizados, estudios clínicos, literatura médica, datos de imágenes (rayos X, MRI), y resultados de tratamientos previos.

## 4. Uber - Precios Dinámicos y Ruteo
- **Nombre del producto/servicio**: Sistema de precios dinámicos y optimización de rutas de Uber.
- **Qué problema resuelve con ML**: Ajusta precios en tiempo real según demanda y optimiza rutas para conductores, reduciendo tiempos de espera y maximizando eficiencia en transporte urbano.
- **Qué tipo de ML usa y por qué**: Por refuerzo, porque aprende a optimizar decisiones en entornos dinámicos como tráfico, maximizando recompensas como tiempo mínimo o ganancias a través de simulaciones y pruebas. Combina con supervisado para predicciones de demanda.
- **Qué datos probablemente necesita para funcionar**: Datos de GPS y ubicación en tiempo real, historial de viajes, patrones de demanda (hora, clima), datos de tráfico, y retroalimentación de usuarios sobre tiempos y calificaciones.

## 5. Nubank - Detección de Fraude y Scoring de Crédito
- **Nombre del producto/servicio**: Plataforma bancaria digital de Nubank (con ML en fraude y crédito).
- **Qué problema resuelve con ML**: Detecta transacciones fraudulentas en tiempo real y evalúa riesgos crediticios para ofrecer préstamos personalizados, protegiendo a usuarios en un ecosistema financiero digital.
- **Qué tipo de ML usa y por qué**: Supervisado, ya que modelos de clasificación se entrenan con datos etiquetados de transacciones pasadas (fraudulentas o legítimas) para predecir riesgos. Es supervisado porque usa ejemplos conocidos para clasificar nuevos datos con precisión.
- **Qué datos probablemente necesita para funcionar**: Historial de transacciones de usuarios, patrones de comportamiento (dispositivos, ubicaciones), datos financieros (ingresos, gastos), y variables de riesgo como montos inusuales o velocidades de transacción.
